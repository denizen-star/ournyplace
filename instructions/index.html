Instructions: Tour & Toured

Manage | Shortlist

Peter and Kerv use Tour to schedule a visit and Toured to capture what you noticed on the walk-through. Scoring (0–5) stays on the Peter and Kerv tabs; this flow is separate.

1. Schedule the visit

  1. Open apartment details: From the shortlist, open any listing (/details/?id=<id>).
  2. Tour tab: Set Visit time (datetime), Scheduling notes (access, parking, who to meet), and During-tour notes if you want free-form jotting there.
  3. Save tour. The shortlist Next actions calendar uses this visit time as the tour anchor.

2. Add to Google Calendar (optional)

  1. After a valid visit time is saved, the Add to Google Calendar link appears on the Tour tab.
  2. Opens a prefilled Google event (~30 minutes) with guests leacock.kervin@gmail.com and peterpapapetrou1@gmail.com. The description includes:
    • Link to the toured checklist page for this listing
    • Summary (location line, rent, beds/baths/sq ft)
    • Scorecard (Avg · Kerv · Peter % and per-criterion 0–5 / N/A)
    • External listing URL when set
    • nyhome details link (/details/?id=…)

3. Fill the toured checklist

  1. Open the form: Go to /details/toured?id=<id> or use details Toured tab → Open toured checklist.
  2. Peter vs Kerv: Tap the partner at the top. Each person has their own answers; nothing overwrites between partners.
  3. For each prompt, use Yes, No, or N/A. Some rows offer chips (noise level, stove type, etc.). Use + for optional notes per row.
  4. At the bottom, add free-form tags (comma of ideas works; use Add).
  5. Save. Data is stored on the listing as JSON (toured_data).

4. Read-only summary on details

  1. On /details/?id=…, open the Toured tab.
  2. You see a table: prompt in the left column, Peter and Kerv in the next columns. Rows appear only when at least one person answered that prompt. Tags show in two columns when present.
  3. After saving changes on /details/toured?id=…, reload or revisit details so the summary reflects the latest data (browser cache may show an older snapshot briefly).

Database setup (one-time per environment)

If checklist Save fails with a message about a missing database column, run against the same database as your app: 

npm run migrate
