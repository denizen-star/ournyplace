# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev        # Start local dev server at http://localhost:8889 (requires Netlify CLI)
npm run migrate    # Schema + seeds, alters (`listing_star`, `toured_data`, `nyp_visits.scheduling_notes` / `toured_notes`, etc.)
npm run split-badges  # Split status badge sprite sheet into individual PNGs
```

Environment: copy `.env.example` to `.env.local` and set `DATABASE_URL` (PlanetScale HTTP connection string) and `APP_NAME=nyhome`.

## Architecture

**nyhome** is a private PWA for two partners (Kerv and Peter) to score and track NYC apartment listings. No build step ‚Äî vanilla JS served directly by Netlify CLI in dev, with Netlify Functions as the API layer and PlanetScale MySQL for persistence.

### Tour & Toured checklist ‚Äî how to use

1. **Schedule the visit:** On **`/details/?id=`** ‚Üí **Tour** tab, set **Visit time**, **Scheduling notes**, and **During-tour notes** as needed, then **Save tour**. Shortlist **Next actions** calendar uses **`visit_at`** from **`nyp_visits`** (`next_visit`).
2. **Google Calendar:** After a valid visit time is saved, use **Add to Google Calendar**. The event is **30 minutes**, invites **leacock.kervin@gmail.com** and **peterpapapetrou1@gmail.com**, and the description includes the **toured checklist link**, listing **summary**, **scorecard** lines, **external listing URL** (if set), and **nyhome details** URL.
3. **Capture on-site / after:** Open **`/details/toured?id=<listingId>`** (or **Toured** tab ‚Üí **Open toured checklist**). Switch **Peter** vs **Kerv** to record separate POVs; tap **‚úì / ‚úó / ‚Äî**, optional **chips**, **+ note**, and **tags**; **Save** persists **`nyp_apartments.toured_data`**. Requires **`npm run migrate`** so **`toured_data`** exists; if saves fail with a migrate hint, run migrate against **`DATABASE_URL`**.
4. **Review at a glance:** On **`/details` ‚Üí Toured** tab (not the standalone URL), see a **read-only** **Prompt ¬∑ Peter ¬∑ Kerv** table: only prompts **with at least one answer** appear; tags show in two columns. Refresh or reload **`/details`** after saving on the checklist page to see updates (cache may show stale apartment until refetch).

Printed guide for operators: **`/instructions`** (`instructions/index.html`).

### Database

All tables prefixed `nyp_`. Money stored as integer cents. Key tables: `nyp_apartments` (**`listing_star`**; **`listing_scores_complete_email_sent`**; **`toured_data`** MEDIUMTEXT nullable ‚Äî toured checklist JSON), `nyp_criteria` (soft-delete: **`active = FALSE`**, **`sort_order = 99`**; app lists **`active = TRUE`** only), `nyp_ratings` (one row per apartment √ó partner √ó criterion, unique; `score` nullable ‚Äî `NULL` = N/A), **`nyp_visits`** (`visit_at`, **`scheduling_notes`**, **`toured_notes`**, legacy `notes`, `outcome`; upserted per apartment), `nyp_applications`, `nyp_neighborhoods` (autocomplete seed data), `nyp_listing_events` (append-only status and vote events; last 50 per apartment on `GET /api/apartments`), **`nyp_building_blacklist`** (unique `normalized_key`; warn-on-save at building level).

### Scoring

Each partner can vote **0‚Äì5** or **N/A** on each active criterion. **N/A** is stored as `NULL` and is **omitted** from that partner‚Äôs weighted average (only criteria with a numeric score contribute weight). Criteria have weights. Per-partner % = `(Œ£ score√óweight / Œ£ included weights) √ó 20` (0‚Äì100 scale). **Avg** = mean of Kerv and Peter when both are non-null.
