# ROI (Return on Investment) Analysis
Perform a cold, hard review of the code changes from a cost-vs-value perspective.

## The Breakdown:
- **Development Cost**: Estimate the complexity and "technical debt" added.
- **Maintenance Risk**: Will this be hard to support in 6 months?
- **Business Value**: Does this directly lead to revenue or user happiness?
- **Efficiency Check**: Could we have achieved 80% of this result with 20% of the code?
- **Gains**: Quantify What did we gain with this change. Performance, UX, feature availability, what?

## Output Format:

### 💰 Investment
- [Code Complexity Score: 1-10]
- [Maintenance Burden: Low/Med/High]

### 📈 Return
- [Primary Benefit]
- [Secondary Gains]

### ⚖️ Final Verdict
- Should we keep this implementation, or is there a leaner way to "ship" this value?
